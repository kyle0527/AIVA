## 🤝 貢獻指南

### **如何參與開發**

#### **1. 選擇開發模組**
- **Go開發者**: 推薦參與認證檢測模組
- **Python安全專家**: 推薦參與密碼學檢測模組
- **滲透測試專家**: 推薦參與後滲透模組

#### **2. 開發環境設置**
```bash
# Python環境
cd services/features/function_crypto  # 或 function_postex
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Go環境
cd services/features/function_authn_go
go mod download
go build ./cmd/worker
```

#### **3. 開發流程**
1. **Fork並Clone**: 複製專案到本地
2. **創建分支**: `git checkout -b feature/module-enhancement`
3. **實現功能**: 按照架構指南開發
4. **測試驗證**: 確保代碼品質
5. **提交PR**: 描述修改內容和測試結果

### **代碼標準**
- **架構一致性**: 遵循已建立的模組架構
- **類型安全**: Python使用型別提示，Go使用嚴格型別
- **錯誤處理**: 完善的異常處理機制
- **文檔完整**: 函數和類的完整文檔
- **測試覆蓋**: 至少80%的代碼測試覆蓋率

### **評審流程**
1. **自動化檢查**: CI/CD自動測試
2. **代碼評審**: 資深開發者評審
3. **安全審核**: 安全團隊評審
4. **整合測試**: 與其他模組整合測試
